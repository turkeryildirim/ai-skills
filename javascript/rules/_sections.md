# Sections

This file defines all sections, their ordering, impact levels, and descriptions.
The section ID (in parentheses) is the filename prefix used to group rules.

---

## 1. TypeScript Types (ts)

**Impact:** CRITICAL
**Description:** TypeScript's advanced type system enables compile-time safety, prevents runtime errors, and creates self-documenting APIs. Covers generics, conditional/mapped/template literal types, utility types, and type-safe architectural patterns.

## 2. Backend Architecture (backend)

**Impact:** HIGH
**Description:** Node.js backend patterns for building scalable, maintainable server applications with Express/Fastify. Covers layered architecture, dependency injection, middleware, error handling, database integration, authentication, caching, and API design.

## 3. Testing (test)

**Impact:** HIGH
**Description:** Comprehensive testing strategies using Jest and Vitest for unit, integration, and E2E tests. Covers mocking patterns, test fixtures, React component testing, and coverage strategies for reliable test suites.

## 4. Async Patterns (async)

**Impact:** CRITICAL
**Description:** JavaScript asynchronous programming patterns including Promise combinators and async/await best practices. Proper async handling prevents data loss, unhandled rejections, and race conditions.

## 5. Functional Programming (func)

**Impact:** HIGH
**Description:** Functional programming patterns in JavaScript including array methods, function composition, currying, memoization, pure functions, and immutability for writing composable, testable code.

## 6. ES6+ Features (es)

**Impact:** MEDIUM
**Description:** Modern JavaScript features including destructuring, spread/rest, template literals, optional chaining, nullish coalescing, generators, and ES modules for writing clean, expressive code.

## 7. Performance (perf)

**Impact:** MEDIUM
**Description:** Performance optimization patterns including debounce, throttle, and lazy evaluation for building responsive, efficient JavaScript applications.
